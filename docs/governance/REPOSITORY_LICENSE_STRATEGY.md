# Repository License Strategy

This document makes the current repository licensing position explicit without
selecting a public license or changing ownership.

## Current Position

No repository license has been selected yet.

The project remains owner-controlled, and any future licensing decision must be
approved by the project owner before it is applied to the repository.

## Decision Principles

Any future license choice should preserve:

- project ownership;
- contributor assignment clarity;
- source-rights caution;
- and the ability to keep the prototype scope narrow.

## What This Is Not

This document is not:

- a license grant;
- a public commitment to open-source distribution;
- or a waiver of ownership.

## Review Requirement

Before any license is added or changed, the owner should review:

- intended distribution;
- contributor expectations;
- commercial goals;
- and whether any third-party material changes the licensing posture.
