# Accessibility Supports

This document defines the accessibility scaffolding for the Texarkana 1885
prototype.

## Purpose

Accessibility must be built into mission design and the student experience, not
bolted on later as an afterthought.

## Required Support Areas

The prototype should keep room for:

- 504 supports;
- Special Education supports;
- English Learner supports;
- vocabulary coaching;
- translation;
- scaffolds;
- teacher accommodations;
- and embedded mission supports.

## Contract Rules

- Accessibility supports remain teacher-controlled.
- The repository must not infer private student needs from mission data alone.
- Translation, scaffolds, and accommodations should be visible as mission-aware supports.
- Accessibility support scaffolding must not alter the historical fact / inference / fiction boundary.

## Current Texarkana Status

The current repo seeds the accessibility-support contract only. It does not yet
implement student-level accommodation profiles or district accessibility
integrations.
