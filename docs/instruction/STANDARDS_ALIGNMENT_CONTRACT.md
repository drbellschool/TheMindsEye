# Standards Alignment Contract

This document defines how standards alignment should be represented before a
mission reaches classroom use.

## Purpose

Standards alignment must be explicit, reviewable, and teacher-controlled.

The system should never imply that a mission is standards-aligned simply
because it mentions writing, history, or problem solving.

## Required Alignment States

Instructional alignment records should distinguish at least:

- HQIM framework seeded;
- TEKS pending teacher selection;
- reviewed subset available;
- and approved for mission use

This allows the repo to make progress without inventing exact standards codes
too early.

## Required Alignment Record Fields

Each instructional alignment record should identify:

- framework;
- subject area;
- grade band;
- alignment status;
- standard identifier when reviewed;
- teacher-facing alignment label;
- evidence expectations;
- teacher-review requirement;
- and notes.

## TEKS Rule

Exact TEKS identifiers should not be invented.

If the teacher has not yet selected the final course, grade, and mission
standard, the contract should say so explicitly.

That means the repo may hold:

- a pending TEKS selection state

without publishing a fake code.

## HQIM Rule

HQIM expectations may be seeded before final TEKS selection.

This allows the mission architecture to preserve:

- evidence use;
- teacher authority;
- student artifact expectations;
- and knowledge-building flow

even while the final standard code is still under review.

## Promotion Rule

A mission should not be treated as classroom-ready solely because an alignment
record exists.

The teacher still needs to confirm:

- final standards target;
- artifact expectations;
- and whether the historical evidence is sufficient for the lesson.

## Current Texarkana Status

The current repo seeds:

- HQIM framework expectations for the placeholder mission;
- and an explicit TEKS pending-selection state

so that future mission approval work has a stable instructional contract.
