# Assessment Artifact Framework

This document defines the narrow assessment-evidence contract for the Texarkana
1885 prototype.

## Purpose

Assessment in The Mind's Eye must remain explainable and teacher-controlled.
The system should support student artifacts without turning evidence review into
a hidden scoring engine.

## Required Rules

- Student artifacts must connect back to the mission, source notes, and map-linked evidence.
- Teachers must remain able to select, upload, or define the rubric.
- Automated comparison may assist review, but it must not override teacher judgment.
- Mastery should stay on a 1-4 scale with a transparent interim conversion path.
- Incomplete should only mean the student did nothing.

## Required Artifact Types

The prototype should support artifact categories such as:

- editorial;
- letter;
- historical report;
- mission reflection;
- and other teacher-approved mission outputs.

## Evidence Packet Expectations

An assessment evidence packet should include:

- the mission ID;
- the expected artifact type or types;
- the visible evidence trail used by the student;
- the mastery scale;
- the teacher override rule;
- and the rubric boundary.

## Current Texarkana Status

The repo currently seeds the assessment-evidence contract only. It does not yet
implement student uploads, rubric authoring, or gradebook integration.
